DATA FLOW
=========

User Request â†’ Web UI â†’ ML Engine â†’ Database â†’ Results â†’ Web UI â†’ User

Detailed Flow:
1. User enters symptom in search box (web_ui.py)
2. Dash callback receives input (web.py)
3. Symptom expanded with synonyms (medimatch_ml.py)
4. TF-IDF vectorizes query (medimatch_ml.py)
5. Cosine similarity computed (medimatch_ml.py)
6. Top medicines returned (medimatch_ml.py)
7. Results formatted as cards (web_ui.py)
8. Analytics graph generated (medimatch_ml.py)
9. Gemini AI advice fetched (medimatch_ml.py)
10. Complete UI rendered (web_ui.py)

INTEGRATION POINTS
==================

When ML Team adds new feature:
1. Add function to medimatch_ml.py
2. Export function in module
3. Inform Web Team about:
   - Function name
   - Input parameters
   - Output format
4. Web Team creates UI component in web_ui.py
5. Integration in web.py (main app)

When Web Team adds new feature:
1. Add component to web_ui.py
2. Add styles to STYLES or CUSTOM_CSS
3. Inform ML Team if data needed
4. Integration in web.py (main app)

IMPORTANT NOTES
===============

1. DO NOT modify web.py directly for team work
   - Use medimatch_ml.py for ML changes
   - Use web_ui.py for UI changes
   - web.py combines both for deployment

2. The datasets are large (250MB total)
   - Use Git LFS for versioning
   - Don't commit without LFS

3. API Keys are in the code
   - Gemini API: In medimatch_ml.py
   - Google OAuth: In web_ui.py
   - Keep them secure!

4. Hugging Face requires port 7860
   - Don't change the port in web.py

5. Python version: 3.11
   - Specified in Dockerfile
